# Ralph-to-Ralph Archive

This directory contains the artifacts from the original autonomous build that created this codebase. It's preserved as a reference and proof-of-concept — you can delete it entirely without affecting the app.

## The Story

At **Ralphthon Seoul 2026**, we pointed the Ralph-to-Ralph pipeline at [resend.com](https://resend.com) and walked away from our computers for 5 hours.

When we came back, we had a fully working Resend clone — deployed to AWS, real emails landing in inboxes, 52 features across 10 dashboard pages, 388 passing unit tests, 102 autonomous git commits. We wrote zero lines of application code. The loops did everything.

> **2nd Place Winner** — Ralphthon Seoul 2026

## How It Was Built

Three autonomous loops ran in sequence, orchestrated by a watchdog:

### Phase 1: Inspect (`inspect-ralph.sh`)
Ever CLI + Claude Opus analyzed resend.com — navigating every page, screenshotting every UI state, reading the docs and API reference. Output: `prd.json` (product requirements) + `build-spec.md` (technical spec).

### Phase 2: Build (`build-ralph.sh`)
Claude Agent worked through `prd.json` feature by feature — writing API routes, React components, DB schema, unit tests, and AWS infrastructure. Each iteration committed exactly one feature. Output: the entire `src/` codebase.

### Phase 3: QA (`qa-ralph.sh`)
Codex agents ran E2E tests via Ever CLI, found bugs, fixed them, and retested. Up to 5 bug-fix cycles ran automatically before declaring done.

`ralph-watchdog.sh` orchestrated all three phases with auto-restart on failure.
